# AGENTS.md — Qt MCP SDK Project Guide

## Project Overview

A Qt5-based MCP (Model Context Protocol) SDK providing client and server libraries for integrating MCP protocol communication into Qt applications.

## Build System

### Build Commands (Shadow Build — Recommended)

All build artifacts (Makefiles, intermediate .obj files, final .dll/.exe) are output to the `build/` directory. **Source directories remain completely clean**:

```bash
# Enter the build directory
cd build

# Generate Makefiles (Debug)
qmake ../qt-mcp-sdk/qt-mcp-sdk.pro -spec win32-msvc "CONFIG+=debug"

# Compile
nmake

# Full cleanup
nmake distclean
```

### Output Layout

All library files (`.dll`/`.lib`) and executables go to `build/bin/`, eliminating runtime dependency copy issues.

- `build/bin/mcpcore.dll` / `mcpcore.lib`
- `build/bin/mcpclient.dll` / `mcpclient.lib`
- `build/bin/mcpserver.dll` / `mcpserver.lib`
- `build/bin/simpleserver.exe`

### Dependency Graph

```
simpleserver (exe)
    ├── mcpclient (dll) ──┐
    └── mcpserver (dll) ──┤
                           └── mcpcore (dll)  ← lowest level, no dependencies
```

### Build Order

`mcpcore` → `mcpclient` + `mcpserver` (can be parallel) → `examples`

## Inter-library Linking Convention

### Dependency Reference Format (in .pro files)

```qmake
# Link dependency library
LIBS += -L$$DESTDIR -lmcpcore

# Runtime dependency (ensures build order)
PRE_TARGETDEPS += $$DESTDIR/mcpcore.lib

# Header path
INCLUDEPATH += $$PWD/../mcpcore
```

### common.pri Shared Configuration

`common.pri` centrally manages shared configuration for all subprojects:

```qmake
# Unified output directory
DESTDIR = $$PWD/../build/bin

# C++ standard
CONFIG += c++17

# Reduce duplication
CONFIG += warn_on
```

## Development Conventions

### Code Style
- Use C++17 standard
- Follow Qt naming conventions (CamelCase class names, file names matching class names)
- Use `#pragma once` or traditional `#ifndef` include guards in headers
- Export macros use `Q_DECL_EXPORT`/`Q_DECL_IMPORT` defined in each module's `_global.h`

### MCP Protocol Design (To Be Implemented)
1. **mcpcore**: Define core MCP protocol data structures, JSON-RPC message types, transport layer abstract interface
2. **mcpclient**: Based on mcpcore, implement client connection management, request sending, response handling
3. **mcpserver**: Based on mcpcore, implement server listening, request routing, tool/resource registration
4. **examples**: Demonstrate how to build MCP applications using the SDK

### Implementation Priority
1. **mcpcore** — Implement MCP protocol core:
   - `MCPMessage` / `MCPRequest` / `MCPResponse` data structures
   - JSON-RPC 2.0 message serialization/deserialization
   - Transport layer abstract interface (`MCPTransport`)
2. **mcpserver** — Implement server:
   - `MCPServer` class managing service lifecycle
   - Tool registration mechanism
   - Resource management
   - Transport implementation based on `QLocalServer` / `QTcpServer`
3. **mcpclient** — Implement client:
   - `MCPClient` class managing connections and sessions
   - Send requests, receive notifications
   - Auto-reconnect mechanism
4. **examples** — Enhance examples:
   - `simpleserver` demonstrating basic MCP server usage

## FAQ

### Q: DLL not found at runtime?
A: All DLLs are unified under `build/bin/`. Ensure the executable is in the same directory. When debugging with Visual Studio, set the working directory to `build/bin/`.

### Q: Steps to add a new module?
1. Create a module directory under `qt-mcp-sdk/`
2. Create a `.pro` file including `common.pri`
3. Create `xxx_global.h` with export macros
4. Add the new module to `SUBDIRS` in the top-level `qt-mcp-sdk.pro`
5. If other modules depend on it, add `LIBS` and `PRE_TARGETDEPS`
